Model system v2 — one owner per job

Supersedes the picker sections of the first draft. Changes after review: the sidebar's "Active model" box is removed (the composer pill is now the single switch); "Show thoughts" is dropped; the thinking state reuses the existing ThinkingIndicator animation (three bouncing dots, --syflo-think-a/-b) instead of inventing a new one.

Division of labour: the composer pill switches between models that are installed and ready — nothing else. Settings manages the library — hardware recommendation, downloads with progress, removal, provider/API-key. Neither place repeats the other's job: the pill never downloads, Settings never switches.

04 Thinking state — same animation as today
The three bouncing dots ARE the existing ThinkingIndicator (same size, delays, easing and theme colors --syflo-think-a/-b) — with thinking on, the rotating tip or quote line simply appears beneath them. No lightbulb icon, no "Show thoughts". Dots and tip live inside the assistant's .prose container — the same box the answer streams into, so nothing jumps. In the default theme that container is invisible (as today); themed modes frame it as their AI bubble automatically (see the gallery). After the answer the block collapses to "Thought for 1m 42s".

05 Outlook — multiple providers configured
Once OpenAI or Claude keys are set, the same picker simply grows group headers — the active selection becomes a (provider, model) pair. Cloud models are always ready (no download concept, no hint rows); the Thinking row appears only when the active model supports it (Ollama think, OpenAI reasoning_effort, Claude extended thinking). Settings gains one section per provider: Ollama keeps the library, cloud providers hold an API key and a curated model list. Nothing else changes.
