Context cards — readability variants

The ancestor summary cards in the ParentContextPane currently render the LLM-written summary as plain text, so Markdown emphasis (*…*) and LaTeX ($dS/dt = f(S, t)$) show up as raw symbols, and the summary reads as one unbroken wall of text. Three variants below, ordered by implementation cost. Base card styling is 1:1 from mockup-chat-highlights-ask-in-chat.html, section 04.

00 Current state (for contrast)

Raw $…$ and *…* markers, one 10-line paragraph. This is the bug report.

Handle arbitrary time scales summary Handle arbitrary time scales
What the model inherits

"Handle arbitrary time scales" is a profound advantage of Neural ODEs, contrasting discrete models like Transformers that pre-bake static snapshots with continuous equations ($dS/dt = f(S, t)$). Discrete models must guess or interpolate between frames, leading to drift over long horizons and errors during rapid events. Conversely, Jemma with Neural ODEs acts like a live physics simulation, solving infinitesimal time steps ($dt$) without accumulation of catastrophic errors. This allows seamless adaptation to variable environments, simulating processes from nanoseconds to hours with one model. For robotics, this ensures safety by eliminating blind spots between steps and allows resource efficiency across different speeds. Unlike Transformers relying on statistical correlations, Neural ODEs maintain physics compliance, transforming robots from frame-by-frame guessers into true physics engines capable of handling arbitrary time scales natively.

01 Variant A — rendered & paragraphed

Frontend-only, smallest change. Pipe the summary through the same Markdown + KaTeX renderer the chat bubbles already use (fixes $ and * for free), and split it into paragraphs at sentence boundaries (~2–3 sentences each). The summary text itself stays exactly what the LLM receives.

02 Variant B — gist + key points

Needs a summary-prompt change (structured output). The summariser returns a one-line gist plus 3–4 bullets with a bolded lead. Scannable in ~5 seconds. Caveat: what the user sees is a re-shaped version of what the model inherits, no longer the literal text — the card's promise ("what the model inherits") weakens.

03 Variant C — two-stage disclosure

Frontend + prompt change. Expanded card shows only a one-sentence gist and key-term chips; "Show full summary" reveals the literal inherited text (rendered as in Variant A). Keeps the card compact even with 3–4 ancestors stacked, and the literal text stays one click away — the "what the model inherits" promise holds.
