How the inline label edit works
  • Each label is a plain text span by default. On hover, a dashed outline appears to hint "this is editable".
  • Click the label → it swaps for a tiny input. The input is the same size, so the popup doesn't reflow.
  • ↩ commits, esc reverts. Clicking outside also commits.
  • Labels are user-global — saved to localStorage (or user settings), used across every paper and view.
  • Colors themselves are fixed — only the names change. This keeps the visual language consistent.
Why no separate "Edit labels" modal? Editing inline is faster, lower friction, and stays in the user's mental flow. Renaming a label is a tiny action — it doesn't deserve a full modal with header, footer and backdrop. The previous edit-labels mockup (mockup-edit-labels.html) is now obsolete.
