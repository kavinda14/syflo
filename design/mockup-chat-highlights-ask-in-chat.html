Chat highlights, "Ask in chat" & the no-PDF branch layout

Three connected surfaces: (01) the PDF highlight interaction extended to chat messages — same right-click popup, same five fixed colors; (02) the "Ask in chat" action, which drops the selection as a removable quote into the SAME chat's composer (no branch); (03) the layout when a branch is opened via "Open as new chat" and no PDF exists — the parent chat takes the center pane as context, the branch opens in the right pane. Popup is 1:1 mockup-popup-edit-labels.html; frame and chat pane are 1:1 mockup-pdf-layout.html.

02"Ask in chat" — quote in the composer

Clicking "Ask in chat" drops the selection into the composer of the chat the user is already in — no new branch — as a compact quote block docked on top of the textarea. Color bar shows the highlight color — chat selections are always auto-highlighted in the active color (decision 2026-07-21), PDF selections stay neutral gray if no color was picked. The source line names where the selection came from (same chat, parent context, or the PDF), and the × removes it. The user types a question below and sends; the message renders the quote as a blockquote above the question.

03No PDF attached — parent chat as center context

When "Open as new chat" creates a branch and the tree has no PDF, the center pane is not wasted: the parent chat moves there as a read-only context view, and the new branch opens in the familiar right pane. The selection that spawned the branch stays visibly highlighted in the parent. With a PDF attached, nothing changes: the PDF keeps the center.

04Inherited context — ancestor chain above the parent chat

A branch inherits the whole path to the root: grandparents and higher as cached summaries, the direct parent verbatim. The center context pane shows exactly that — the chain line on top, one collapsible summary card per ancestor (collapsed by default), then the parent chat sheet unchanged below. What the user sees here is literally what the LLM receives.

Ancestor-context decisions baked into this mockup (2026-07-20)
  • Chain line first: root title, then the branch word of every level down to the current branch — the cheapest "how did I get here" thread. Always visible, never collapsible.
  • One card per ancestor above the parent (grandparents+, root first), collapsed by default; header shows chat title, a summary tag and the branch-word badge. The body is the cached summary — the exact text the LLM receives.
  • The parent chat stays verbatim below the cards, unchanged from section 03 — it is inherited word-for-word, so it is rendered word-for-word.
  • Depth-1 branches show no cards (there are no grandparents), only the chain line above the parent sheet.
  • Lucide icons only, no emojis; standard theme tokens so all five themes inherit correctly.
Behavior decisions baked into this mockup
  • Same gesture as the PDF: select, then right-click. No auto-appearing toolbar on selection. The existing FloatingPopup gains the color row for chat selections (it already exists, currently disabled for chat) plus the "Ask in chat" footer button; "Open as new chat" moves to secondary style below it.
  • Chat highlights are reflow-safe: anchored to message_id + start/end character offset into the message text, rendered as tinted spans — not geometric rects like PDF highlights. Same five fixed colors, same global labels.
  • "Ask in chat" targets the CURRENT chat: it drops the selection as a quote block into the composer of the chat the user is already in — no branch is created. Selections from the PDF or the parent-context pane also land in the active chat's composer. The quote is removable via ×; on send it renders as a blockquote above the typed question and is included in the LLM context. Branching stays "Open as new chat". Both actions auto-highlight a chat selection in the active color when no swatch was clicked (decision 2026-07-21) — for branching that is what keeps the spawning selection visible in the parent (section 03).
  • Center pane logic (branching without PDF): PDF attached → PDF keeps the center (unchanged). No PDF and a branch is active → the parent chat renders in the center as a read-only context view with its highlights visible; the toolbar link jumps back into it. Selecting text in the center context view also works (right-click → same popup).
  • Highlights on user messages force dark text on the blue bubble for contrast.
  • esc closes the popup; removing the quote never deletes the highlight.
