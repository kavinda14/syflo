Best of both worlds: the explicit "Edit labels…" link makes the affordance immediately clear, but instead of opening a separate modal, the same popup transforms in place. The color-picker row swaps out for a vertical list of editable inputs. Click "Done" to swap back.

How the transformation works
  • The popup keeps the same width, position, header and body. Only the color row swaps in place — picker (State A) ⇄ edit list (State B).
  • The header above the color row changes too: "HIGHLIGHT COLOR / Edit labels…" → "RENAME COLORS / Cancel". Consistent location, clear meaning.
  • The footer button swaps: "Open as new chat" → "Done — save labels". The user understands they're in a different mode.
  • Changed labels get a blue border (see "Cite later" — was "Reference"). Reset button activates only when the label differs from its default.
  • esc cancels and reverts. ↩ on the "Done" button saves. Pressing ↩ inside an input moves focus to the next row.
  • Labels are saved globally — once you've renamed "Reference" to "Cite later", every popup in every paper shows "Cite later".
