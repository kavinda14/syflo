"Think" toggle & visible reasoning

qwen3.5 is a reasoning model: it writes a long hidden chain of thought before answering. Default is thinking off (fast answers). A per-message chip in the composer turns it on for hard questions; while it runs, the reasoning streams into a collapsible block so the chat never looks frozen. The chip only appears when the active model supports thinking.

01 Composer chip — off (default) and on
The chip sits left of the textarea, next to the plus button. Off: neutral outline. On: blue tint, stays on for this chat until toggled off. Tooltip explains the trade-off.

Left: the reasoning streams live into an open block — the user sees progress from second one. Right: once the answer is done the block collapses to a single line ("Thought for 1m 42s") and can be reopened.

Why does the attention mechanism scale with the square of sequence length?
Syflo
Attention compares every token with every other token. With n tokens, the model computes an n×n score matrix — doubling the sequence length quadruples both the compute and the memory for that matrix. That is the quadratic scaling.
