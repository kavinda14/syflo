Inherited context — concept variants

User feedback (2026-07-22): the ancestor summary cards at the top of the ParentContextPane don't explain what they are. Nothing communicates that this stack is the background knowledge automatically fed into the current branch chat — it just reads as unexplained "context" floating above the parent conversation. Three concept variants that make the inheritance relationship visible. Copy stays in product English; icons are Lucide.

01 Variant 1 — labeled inheritance block

Smallest change. The cards move into a tinted, framed block with a headline and one explainer sentence that names the receiving chat. The block answers "what is this?" in place; card internals stay as they are.

02 Variant 2 — flow rail, ending at "you are here"

Makes the direction explicit. A vertical rail connects the ancestors top-to-bottom, each connector labeled with how the context travels ("condensed into a summary", "you branched on »word«"), and ends in a highlighted "you are here" node for the current chat. The inheritance is no longer implied — it is drawn.

03 Variant 3 — context belongs to the chat that receives it

Relocation instead of explanation. The cards leave the center pane entirely. The branch chat (right column) gets a slim banner: "Carries background from 2 earlier chats" — tapping it expands the summaries inside the chat column. The center pane shows only the parent conversation. Conceptually cleanest — the context sits where it is consumed — but it changes the layout contract from mockup §04.
